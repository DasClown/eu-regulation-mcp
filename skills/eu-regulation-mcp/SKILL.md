# EU Regulation Intelligence MCP

Early warning system for EU regulation. Monitors EUR-Lex, EU Parliament (OEIL), EU Commission (Have Your Say), ECJ (CURIA), and national gazettes (DE/FR/IT/ES).

**Workflow Principle:** Review content before deployment. Never deploy blindly — check seed data first, then write wiki docs, then set up cron jobs, then test.

## 🚀 GitHub Public Deployment

The entire system is distributed as a public GitHub repo at [github.com/DasClown/eu-regulation-mcp](https://github.com/DasClown/eu-regulation-mcp). Only EU-regulation-specific files are included — no auth files, .env, or DB caches.

## 🛠️ Available Tools

### `track_regulation(keyword, sector, region='EU')`
- Subscribe to a topic (e.g. "Glyphosate", "Plant Protection")
- Returns: current legislative status + next steps + deadlines + source links
- **Always append disclaimer**, even if no cache hit

### `get_legislative_status(celex_number, procedure_number)`
- Where is a law in the process? EP reading? Council? Trilogue?
- Timeline + prognosis + link to OEIL/EUR-Lex
- **If empty results:** provide direct link to EUR-Lex/OEIL

### `get_open_consultations(sector, days_remaining)`
- Open EU Commission consultations
- Relevance score: 5=⚠️ URGENT (<14d), 4=🔔 Soon (<30d), 3=📋 Medium, 2=ℹ️ Low
- Results sorted by deadline

### `get_national_implementation(eu_directive, member_state)`
- How was an EU directive transposed in country X?
- **Important:** Regulations (CELEX: 3xxxRxxxx) do not need national transposition — direct application. Only directives (3xxxLxxxx, 3xxxDxxxx) have transposition deadlines.
- Status: `adopted`, `drafting`, `not_started`, `overdue`

### `get_relevant_rulings(keyword, court='ECJ')`
- ECJ rulings on a topic
- Tenor + relevance for business + link to curia.europa.eu

### `regulatory_impact_assessment(sector, action)`
- Check: is a planned action affected by new/upcoming regulations?
- Example: "I want to sell glyphosate alternatives in DE — what's coming?"
- **Keyword extraction:** Individual words (>4 chars) + bigrams are extracted from the action
- Risk level: 🔴 HIGH (≥4 factors) → 🟡 MEDIUM (≥2) → 🟢 LOW

### `alert_check()`
- Proactive check for changes
- **Detects:** Overdue national transpositions (`status != 'adopted' AND deadline < today`), urgent consultations (<14d), upcoming legislative deadlines (<30d)
- **Note:** For regulations (CELEX type R), "overdue transposition" is incorrect — regulations apply directly. Only directives (type L/D) require transposition.
- Output: Structured list with `type`, `severity` (high/medium), `message`, `detail`, `source`

## 🎭 Have Your Say — Playwright-based brpapi Extraction

The Have Your Say portal (`ec.europa.eu/info/law/better-regulation/have-your-say/initiatives_en`) is an **Angular SPA**. Server-side HTTP clients (curl, requests) only receive the SPA shell HTML. Data is loaded asynchronously from internal `brpapi/` endpoints.

### Workflow (tested 2026-05-07)

1. Install Playwright Chromium: `playwright install chromium`
2. Load SPA page in headless browser
3. Intercept AJAX responses via `page.on('response')`
4. Extract brpapi endpoints:
   - `closingSoon?language=EN` → 12 closing initiatives
   - `searchInitiatives?page=0&size=50&language=EN` → paginated search (4033+ total)
   - `groupInitiatives/{id}` → initiative detail (returns JSON!)
   - `initiativeHighlights?language=EN` → highlighted initiatives

Detail documentation: `references/brpapi-playwright-scraping.md`

### EU SPARQL Collector

`collect_eurlex.py` uses POST to `publications.europa.eu/webapi/rdf/sparql` with URL-encoded SPARQL queries. Returns basic metadata (CELEX, title, type, author, date) for regulations and directives. Limited to 20 results per run to avoid timeout. See `references/eurlex-sparql-ontology.md` for the working ontology properties (cdm:work_title, not cdm:title_legal).

### ECJ CURIA Collector

`collect_ecuria.py` uses Playwright to load the CURIA search page (`curia.europa.eu/juris/liste.jsf`). Since CURIA is a React SPA with no REST API, it attempts to:
1. Search via liste.jsf with query parameters
2. Fall back to DNR (Document not Registered) search
3. If all fail, reports CURIA as unavailable

### Reseed Workflow (Development)
For data corrections: use the reseed script:
```python
#!/usr/bin/env python3
"""Clear DB and reseed"""
import sqlite3, os
db = '/root/.hermes/profiles/eu_regulation/cache/eu_regulation.db'
seed = '/root/.hermes/profiles/eu_regulation/scripts/seed_database.py'
conn = sqlite3.connect(db)
for t in ['eurlex_metadata','legislative_procedures','national_implementation',
          'eu_consultations','ecuria_rulings','tracking_subscriptions']:
    conn.execute(f'DELETE FROM {t}')
conn.commit()
conn.close()
exec(open(seed).read())
```

### CELEX Type Rules (Important)

| CELEX Pattern | Type | Transposition Required? |
|--------------|------|------------------------|
| `3xxxRxxxx` | Regulation | ❌ Direct application. Entry in `national_implementation` only for implementing acts (supervision, penalties) with "Regulation" note. |
| `3xxxLxxxx` | Directive | ✅ Member states must transpose by deadline. Status: `adopted`, `drafting`, `not_started`, `overdue`. |
| `3xxxDxxxx` | Decision | ⚠️ Varies. |
| `5xxxPCxxxx` | Proposal | ❌ Not yet in force. Only in `legislative_procedures`. |

**Pitfall:** `alert_check` detects `status != 'adopted' AND deadline < today`. For regulations (R-CELEX), these alerts are misleading — the deadline refers to national implementing acts, not transposition.

### Cron Prompt Rules
1. **Concrete shell commands** — copy-paste ready, no descriptions
2. **Document API errors** — `HTTP 202 (WAF)`, `404`, `Timeout` are expected
3. **Disclaimer in EVERY output** — `"⚠️ Not legal advice. Status: [TODAY]"`
4. **Structured sections:**
   - 🔴 High-risk Alerts
   - 🟠 Open Consultations (<14d)
   - ⏰ Deadlines < 30 days
   - 📋 Active Trackings
   - 🌾 For crop-mcp / 💊 For drug-pipeline
5. **Empty results:** Brief status message, no empty lists

## ⚠️ Pitfalls & Lessons Learned

### API Limitations
- **EUR-Lex:** CloudFront WAF blocks most programmatic access. SPARQL works for basic metadata queries but not full-text search. Ontology uses `cdm:work_title` (NOT `cdm:title_legal`) — see `references/eurlex-sparql-ontology.md` for the full property reference.
- **OEIL (EU Parliament):** All tested API paths return HTTP 404. API appears decommissioned.
- **OData/Cellar:** Seem discontinued or restructured.
- **Better Regulation API:** Returns HTML only, no JSON.
- **Have Your Say brpapi:** `initiativeDetail/{id}` returns **HTML** — use `groupInitiatives/{id}` for JSON data.

### Data Modeling
- ✅ **Directives (L/D-CELEX)** need national transposition → stored in `national_implementation` table
- ❌ **Regulations (R-CELEX)** do not need transposition → entry in `national_implementation` only for implementing acts, with "Regulation — direct application" note
- ✅ **ECJ rulings** need keywords + summary for searchability
- ✅ **Consultations** need sector tag + summary + relevance_score for filtering

### Disclaimer
- **Always append** — in tool outputs, wiki entries, cron results
- **Standard:** `"⚠️ Not legal advice — automated preliminary check, not a substitute for professional legal counsel. Status: YYYY-MM-DD"`
- **Impact assessment:** Prepend `"⚠️ Automatic risk preview."`

### Known Issues

**closingSoon deduplication:** The endpoint sometimes returns the same initiative twice (e.g., ID 14578 Cybersecurity Act duplicated). Before DB insert, deduplicate per `initiativeId`.

**daysLeft=0 is falsy in Python:** `item.get('daysLeft') or 99` returns `99` for `daysLeft=0`. Always use `item.get('daysLeft', 99)`.

**groupInitiatives/{id} returns JSON (confirmed):** This endpoint (not `initiativeDetail/{id}`) is the correct way for initiative details. Contains `dossierSummary`, `reference`, `unit`, `dg`, `topics`, `foreseenActType`, `stage`, etc.

**Mass English migration pattern:** When rewriting a multi-file system from German to English, delegate small files in parallel (patch), use write_file for full rewrites of large files, then reseed and test. The SPARQL collector's subagent discovered the ontology uses `cdm:work_title` (not `cdm:title_legal`) — always verify ontology property names empirically before committing to a query.
